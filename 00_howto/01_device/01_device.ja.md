# 第1章 — はじめての C99 デバイスを作る

> この章は、IoTMaker を初めて使う方のためのものです。章の終わりには、
> `print_int`、`print_float`、`print_string` という3つのブロックを持つ
> `stdOut` というデバイスを作成し、そのうちの1つをプロジェクトの中で
> 使って C99 のソースコードを生成できるようになります。
>
> 所要時間の目安: 20分。前提知識: C言語の基礎
> (簡単な関数が書けること)。

---

## 1.1 デバイスとは?

IoTMaker における**デバイス**とは、ユーザーが**ステージ**(IDE の作業
エリア)にドラッグして、ワイヤーで他のボックスとつなぐグラフィカルな
箱のことです。それぞれのワイヤーは値を運びます — 数値、テキスト、信号
などです。

すべてのデバイスの裏側には、本物のコードがあります。あなたが
**スペシャリスト**であれば、そのコードを一度書くだけで、IoTMaker が
それをグラフィカルなブロックに変換します。それ以降は、プログラミングを
まったく知らない人でも、ワイヤーをつなぐだけであなたのブロックを使える
ようになります。

必要なのは、コードをコミュニティと共有して、使うことを許可すること
だけです。

この章では、考えうる限り最もシンプルなデバイスを作ります。値を画面に
表示する3つのC関数 — 整数、浮動小数点数、テキスト — です。
**それぞれの関数が、同じデバイスの中で独立したブロックになります。**

## 1.3 Cコードを書く

以下のコードを **Editor** に入力(または貼り付け)してください:

```c
#include <stdio.h>

// print_int writes a single integer to standard output, followed by a
// newline. Host targets only (PC/Linux): stdout is a real stream here.
// On microcontrollers there is no stdout — the serial/UART path is used
// instead, so this function is not portable to embedded targets.
void print_int(int value) {
    printf("%d\n", value);
}

// print_float writes a single floating-point number to standard output,
// followed by a newline. The %f conversion expects a double; a float
// argument is promoted to double automatically by the C varargs rules,
// so passing `value` directly is correct and portable. Host targets
// only (PC/Linux) — see print_int for the embedded-target caveat.
void print_float(float value) {
    printf("%f\n", value);
}

// print_string writes a NUL-terminated string to standard output,
// followed by a newline. A NULL pointer is printed as "(null)" instead
// of being passed to printf — printf("%s", NULL) is undefined behavior
// in C, so the guard keeps the block safe no matter what the maker
// wires into it. Host targets only (PC/Linux) — see print_int.
void print_string(const char *value) {
    if (value == NULL) {
        printf("(null)\n");
        return;
    }
    printf("%s\n", value);
}
```

![Cコードが入力されたエディタ](/00_howto/01_device/img/c99-04-new-device.png)

このコードでは4つの重要なことが起きています。先に進む前に、それぞれを
理解しておきましょう:

**関数がブロックになります。** このファイルには3つの公開関数 —
`print_int`、`print_float`、`print_string` — があるので、`stdOut`
デバイスはユーザーに3つのブロックを提供します。デバイスは工具箱のような
もので、関連するブロックをまとめる役割を持ちます。

**パラメータがポートになります。** 各関数のパラメータは、対応する
ブロックの入力ポートになります — ユーザーはそこに、表示したい値を運ぶ
ワイヤーをつなぎます。パラメータの型は、ワイヤーの色と、接続できる
ポートを決めます:

| Cの型          | IDE のワイヤー型 | 補足                                                                                                                       |
|----------------|------------------|------------------------------------------------------------------------------------------------------------------------------|
| `int`          | `INT`            | 整数。                                                                                                                     |
| `float`        | `FLOAT`          | 浮動小数点数。ビット幅(32/64)はコード生成時にターゲットボードが決めるので、ここで気にする必要はありません。               |
| `const char *` | `STRING`         | テキスト。C では文字列は文字へのポインタです。`const` は、関数がテキストを読むだけで、決して変更しないことを示しています。 |

**コメントがドキュメントになります。** 各関数の直上にあるコメント
ブロックは Parse に取り込まれ、IDE の中でユーザーに表示されます。
コードを*読む*人ではなく、ブロックを*使う*人のことを考えて書いて
ください。
1.8節では、複数の言語でドキュメントを書く方法を見ていきます。

**防御的なコードがユーザーを守ります。** `print_string` の `NULL`
ガードに注目してください。C では、ヌルポインタを `printf("%s", ...)`
に渡すのは未定義動作で、プログラムがクラッシュする可能性があります。
スペシャリストは、メイカーがポートに何をつなぐかをコントロールできない
ので、このブロックは危険を冒す代わりに `(null)` を表示します。防御的な
ブロックは優れたスペシャリストの証です。ユーザーは、この小さな箱が
自分のプログラムを絶対に壊さないと信頼できるのです。

> **ヒント:** このデバイスは `printf` を使いますが、これはオペレー
> ティングシステムを持つターゲット(PC/Linux)にしか存在しません。
> マイクロコントローラには `stdout` がなく、出力はシリアル/UART を
> 通ります。だからこそコメントで、これらの関数が組み込みターゲットには
> 移植できないと警告しているのです。制限事項を文書化するのも仕事の
> 一部です。ブロックのユーザーは、あなたが書いたことを信頼します。

## 1.4 Parse: コードをブロックに変える

コードの準備ができたら、**Parse** をクリックします(画面右側の
リマインダー *Click Parse to visualise* に注目してください)。

Parse はファイルを読み、3つの関数とそのパラメータを検出します。
続いて **Wizard** が、見つかった項目ごとに設定ウィンドウを開きます。
まず `print_int` を詳しく設定しましょう — あとは `print_float` と
`print_string` に同じ手順を繰り返すだけです。

最初のウィンドウは関数のものです:

![Wizard](/00_howto/01_device/img/c99-05-new-device.png)

| フィールド           | 入力する内容                                                                                                                                 |
|----------------------|------------------------------------------------------------------------------------------------------------------------------------------------|
| **ID**               | 内部識別子。関数名から生成され、変更できません。                                                                                             |
| **Label**            | ステージ上のブロックに表示される名前。関数名よりも親しみやすい名前にできます。                                                               |
| **Icon**             | ブロックの上部に表示される絵。グリッドからアイコンをクリックするか、名前を入力します(`bolt` や `gear` などの FontAwesome アイコンです)。    |
| **Callback handler** | **— Not a callback handler —** のままにしてください。コールバックは上級機能で、別の章で扱います。                                            |

遠くからでもユーザーがブロックを見分けられるアイコンを選びましょう —
表示系の関数なら、ターミナルや外向きの矢印のようなものが適しています。
3つのブロックで同じアイコンを使っても、変えても構いません(例えば、
整数に `hashtag`、浮動小数点数に `percent`、文字列に `quote-right`
など)。設定したら、ウィンドウの **Save** をクリックします。

![print_int 関数の Wizard](/00_howto/01_device/img/c99-06-new-device.png)

> Cancel と Save の間にある **Add help** ボタンは、デバイスの
> マニュアルを作成します。1.8節で使うので、今は先に進みましょう。

## 1.5 ポートを設定する

続いて、Wizard は各関数のポート設定を開きます。ウィンドウのタイトル
にはフルパスが表示されます — 例えば
**Port · print_int · input · value**(*print_int* 関数の入力ポート
*value*)です。

ここで最も重要なフィールドは **Connection** で、2つの選択肢が
あります:

- **Optional — port may be left unwired**: ユーザーはポートにワイヤーを
  つながなくても構いません。関数に妥当なデフォルト値がある場合や、
  パラメータが微調整用の場合に使います。
- **Mandatory — port must be wired**: ユーザーはこのポートに必ず
  ワイヤーをつながなければなりません。つながないと、IDE はコード生成の
  前にエラーを報告します。

このデバイスの3つのポートには、**Mandatory** を選んでください。「何も
ない」を表示することには意味がないので、各ポートは常に値を受け取る
必要があります。

**Comment** フィールドも必須です。これはポートの説明で、IDE でユーザー
がピンにマウスを重ねたときに表示され、生成されるコードのコメントにも
なります。短い一文で十分です — ポートが*何を運ぶか*を書いてください。
例えば:

| ポート                 | コメントの例                    |
|------------------------|---------------------------------|
| `print_int · value`    | `Integer value to print`        |
| `print_float · value`  | `Floating-point value to print` |
| `print_string · value` | `Text to print`                 |

![コメントを設定したポート](/00_howto/01_device/img/c99-07-new-device.png)

各ポートのウィンドウで **Save** をクリックします。終わる頃には、
3つの関数と3つのポートの設定が完了しています。

---

### 1.5.1 スカラーポインタ：プローブ・ワイヤ

関数がスカラーへのポインタを**返す**場合（`int32_t *get_buffer()`）、
または `direction:out` なしの**通常入力**として受け取る場合
（`const int32_t *data`）、そのポートは**ポインタ・ワイヤ**を公開します。
色はベース型と同じで、**破線**で描画されます。整数幅はすべて `int*`
ファミリートークンに集約され、`float`/`double` は `float*` になります。

ポインタ・ワイヤは **Debug（Print）デバイスにのみ**接続できます —
プローブ用のワイヤです。生成コードは自動的にデリファレンスし、
ヌルポインタの場合はクラッシュせず `null pointer` と出力します。
既存の三つの規約は維持されます：`char *` は従来どおり C の
**文字列（値渡し）**、構造体ポインタは**ハンドル**
（リソースチェーンのイディオム）、`direction:out` 付きポインタは
従来どおり値の出力に分割されます。

## 1.8 デバイスのマニュアルを書く(推奨!)

デバイスはもう動きますが、使う人にとって一番大切なものがまだ足りま
せん。**マニュアル**です。

マニュアルを書かない場合、IDE はソースコードのコメントだけをユーザーに
表示します — 何もないよりはましですが、エンドユーザーには不要な技術的
詳細が混ざってしまいます。

私たちが目指すのは、きちんとしたマニュアルのある状態です。作成手順:

1. **Editor** に戻り、**Parse** を実行します(ヘルプマネージャは、
   どのファイルを作成できるかを知るためにパース結果を必要とします)。
2. **Wizard** を開き、いずれかの関数のウィンドウに入って
   **Add help** をクリックします。
3. ヘルプファイルマネージャが、作成ウィンドウを準備した状態で開き
   ます。次の項目を選びます:
	- **何を文書化するか:** *main menu text* はデバイスの全体説明
	  (IDE のメニューに表示されるテキスト)を作成します。関数名
	  (例: `print_int`)は、その特定のブロックのドキュメントを作成
	  します。
	- **言語:** `en`、`pt-br`、`es`、`fr`、`de`、`zh-cn`、`ja`。
4. Markdown で内容を書いて保存します。
5. カバーしたいブロックと言語ごとに繰り返します。

![ドキュメント - ステップ1](/00_howto/01_device/img/c99-09-new-device.png)

![ドキュメント - ステップ2](/00_howto/01_device/img/c99-10-new-device.png)

![ドキュメント - ステップ3](/00_howto/01_device/img/c99-11-new-device.png)

ファイルは `<名前>.<言語>.md` という規則に従います。`stdOut` を英語と
ポルトガル語で完全にカバーすると、次のようなセットになります:

```
readme.en.md          readme.pt-br.md          ← メニューのテキスト
print_int.en.md       print_int.pt-br.md       ← int ブロックのマニュアル
print_float.en.md     print_float.pt-br.md     ← float ブロックのマニュアル
print_string.en.md    print_string.pt-br.md    ← string ブロックのマニュアル
```

IDE は、ユーザーの言語に合ったファイルを自動的に選びます。

**ブロックのマニュアルの推奨構成**(ネイティブデバイスが使っているのと
同じものです):

```markdown
# print_int — 整数を表示

整数を画面に表示し、改行します。

## ポート

| ポート | 方向 | 型  | 説明           |
|--------|------|-----|----------------|
| value  | 入力 | int | 表示する値     |

## ヒント

このデバイスは PC/Linux ターゲットでのみ動作します。マイクロ
コントローラでは、シリアル出力デバイスを使ってください。
```

## 1.9 プロジェクトでデバイスを使う

いよいよ `stdOut` を実際に使ってみましょう。IDE で **C99** タイプの
プロジェクトを作成(または開く)します — ステージ右上の `C99`
インジケーターに注目してください。

**Menu**(ステージの隅にある六角形)を開き、サイドバーの
**My Items** に移動します。ここには*あなたが*作ったデバイスが並び
ます。`stdOut` の中に3つのブロックが見つかります:

![メニュー - ステップ1](/00_howto/01_device/img/c99-12-new-device.png)

![メニュー - ステップ2](/00_howto/01_device/img/c99-13-new-device.png)

`print_int` をクリックし、**+ Place on stage** を押します。ブロックが
ステージに現れ、ワイヤーを受け入れる準備が整います。

テストとして、次の小さなプログラムを組み立てましょう:「数値のリストの
インデックス3の要素を取り出して、画面に表示する」。あと3つ、すべて
IDE ネイティブのデバイスが必要です:

- **Const → Array Int**(`constArrayInt`): リスト `{0,1,2,3,4,5,6,...}`。
- **Const → Int**(`constInt`): 定数 `3`。取り出したいインデックス
  です。
- **Index**(`indexInt`): リストとインデックスを受け取り、要素を
  返します。

ワイヤーを次のようにつなぎます:

1. `constArrayInt_0` の出力 → `indexInt_0` の1番目の入力(リスト)。
2. `constInt_0` の出力 → `indexInt_0` の2番目の入力(インデックス)。
3. `indexInt_0` の出力 → あなたの `print_int_1` の **value** ポート。

![ide](/00_howto/01_device/img/c99-14-new-device.png)

ワイヤーは、互換性のある型のポート同士しか「つながらない」ことに注目
してください — リスト(`[]INT`、太いワイヤー)はリスト用ポートにしか
入らず、普通の `INT` は `INT` ポートにしか入りません。IDE は、コードを
生成する前の段階で、間違った接続を防いでくれます。

> **試してみましょう:** `print_int` を `print_string` に置き換えて、
> index の `INT` 出力をそのポートにつないでみてください。IDE は接続を
> 拒否します — 型が合わないからです。C ではコンパイル時(最悪の場合は
> 実行時)にしか現れないエラーを、IoTMaker はこうして組み立ての時点で
> 防いでいるのです。

## 1.10 ソースコードを生成する

![コードエクスポート - 1](/00_howto/01_device/img/c99-15-new-device.png)

![コードエクスポート - 2](/00_howto/01_device/img/c99-16-new-device.png)

![コードエクスポート - 3](/00_howto/01_device/img/c99-17-new-device.png)

プログラムが組み上がったら、**Menu → Export** を開いて C コードの
エクスポートを選びます。IDE は、どのボード向けにコードを生成するかを
尋ねます — このテストでは、自分のコンピュータでコンパイルして実行
できるように **PC** を選んでください(`printf` はマイクロコントローラ
には存在しないことを思い出してください)。

IoTMaker は、完全な C99 プロジェクトを zip ファイルとして生成します。
中身は次のとおりです:

- あなたの `stdOut` デバイスのコード。書いたままの形で入っています。
- 使用したネイティブデバイス(定数、index)のコード。
- ステージ上のワイヤーに従って、すべてを正しい順序でつなぐ `main`。

解凍し、コンパイルして、実行してください。すべてうまくいけば、
プログラムは `3` を画面に表示します — リスト `{0,1,2,3,...}` の
インデックス3の要素です。

**おめでとうございます! はじめての C99 デバイスを作成し、それを
使って本物のプログラムを生成できました。** 🎉

## 1.11 よくあるエラー

| 症状                                                     | 考えられる原因                                                             | 解決策                                                                                             |
|----------------------------------------------------------|------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------|
| Parse が関数を見つけない                                 | 関数のシグネチャが不完全か、ファイルに構文エラーがある                     | Editor でコードを確認してください。**Live analysis** が入力中に問題箇所に下線を引きます           |
| **Add help** ボタンが "Run Parse first" と表示する       | ヘルプマネージャはパース結果を必要とします                                 | **Parse** を実行して、もう一度試してください                                                      |
| ブロックが **My Items** に現れない                       | Parse の後にデバイスが保存(**Save**)されていない                           | デバイスのエディタに戻り、バージョンを保存してください                                            |
| IDE がワイヤーのない必須ポートを報告する                 | **Mandatory** のポートがステージ上で未接続のまま                           | 指摘されたポートにワイヤーをつなぐか、デバイスの次のバージョンでポートを **Optional** にしてください |
| ワイヤーがポートにつながらない                           | 型に互換性がない(例: `INT` を `STRING` ポートへ)                           | 変換ブロックか、正しい型のポートを使ってください — `stdOut` には型ごとのブロックがあります        |
| ブロックのドキュメントが「混ざって」見える(2言語)        | デバイスにマニュアルファイルがなく、IDE がソースコードのコメントにフォールバックした | マニュアルファイル(1.8節)を言語ごとに作成してください                                             |

---

## 重要なヒント

`export` > `image` メニューから、プログラムの画像を保存できます。この
画像は、編集されていない限り、中にソースのコピーを持っています —
IDE のステージにドラッグするだけで、読み込み直せます。

ただし、あなたが作成したデバイスは画像に含まれません。それらは
ユーザーのライブラリに存在している必要があります。
